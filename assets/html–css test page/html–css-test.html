Basic block elements.

This is how a normal paragraph (p element) look. This page was primarily written for testing the effect of user style sheets. You can use it for various other purposes as well, like just checking how your browser displays various HTML elements by default. It can also be useful when testing conversions from HTML format to other formats, since some elements can go wrong then.

In this other paragraph, I would like to shout out to Jukka Korpela who writes the original test document in which this document is based on. I decided to rewrite the document to cover the needs of HTML5 semantics and structure which is not yet covered—at least according to my personal needs.

Unlike other parts that wrapped in section element, this Basic block elements part is wrapped in article for testing purpose. Article is a self-contained, standalone section of an HTML document in which its content should make sense on its own and should be possible to distribute it independently from the rest of the site (e.g. blog post).

The following is preformatted text. Preformatted text is defined with pre and is an unusual HTML tags that takes notice of every character in it (as illustration, other elements will ignore a consecutive space or a line-break). It is most commonly used for blocks of code, where spacing, such as indentations, can be relevant.

        This     tag     will     render
      the     consecutive     spaces,
    the     indents,     and
  all     the     line-breaks
    exactly     as     how
      it     is     written
        in     the     HTML     doc.

Another basic block element that's worth mentioning is address. address element should only used to specifies contact informations of the entire web page, or or to an article element. Hence, it usually only used once and placed at the top or bottom of the document. Check the footer section of this document to see how it looks.

Text-level markups

  • CSS; this is a sample of abbr, a tag that you use for abbreviation, a shortened form of a word or phrase. Keep in mind that accronym element is not supported in HTML5 and you should use abbr instead.
  • See this; this is a sample of em, a tag that you use to stress a word or phrase in a sentence.
  • Attention here; this is a sample of strong, a tag that you use to highlight part of a sentence that matters most, a warning, or something the user should act on quickly, etc.
  • HTML5 is the thing!; this is a sample of mark inside of a quotation, a tag that you use to highlight a word or phrase that the original source/author did not intend to emphasize.
  • Lihat sini; this is a sample of i, a tag that you use to italic a word or phrase that is not for emphasizing, but only to visually present text of different quality. This tag is usually used for foreign words or latin names.
  • Unimportant bold; this is a sample of b, a tag that you use to bold a word or phrase that is not for emphasizing, but only to visually present text of different quality. This tag is almost unneeded but part of HTML5 spec nonetheless.

Lists

This is a paragraph before an unnumbered list: (ul). Note that the spacing between a paragraph and a list before or after that is hard to tune in a user style sheet. You can't guess which paragraphs are logically related to a list, e.g. as a "list header".

  • One.
